## Key Decisions Made

- **Stack:** Angular 18+ / Angular Material · NestJS · Prisma · PostgreSQL — mirrors budgetwise for skill/knowledge reuse.
- **Auth:** JWT email/password for Dash accounts. Reddit OAuth is only for linking individual `RedditAccount`s, not for logging into Dash.
- **No wiki, no tickets.** Prompt-driven workflow with `PROJECT-STATUS.md` as the only persistent memory.
- **Reddit client:** plain `axios` against `oauth.reddit.com`. `snoowrap` is un-maintained; only reach for it if it saves real work.
- **Reddit auth = one shared Devvit token, not per-account OAuth.** A registered Reddit "web app" (client_id/secret) is no longer obtainable, so Dash reuses the token minted by `devvit login` (`~/.devvit/token`) — a standard Reddit OAuth2 refresh token, `scope=*`, from Reddit's first-party **public** Devvit client (no secret). Since comment history is public, one such token reads every tracked account by username, so accounts are tracked by username only (no per-account link/consent/token). Refresh must use the same Devvit client that minted the token: copy-paste login → `TWTsqXa53CexlrYGBWaesQ` (the code default), localhost login → `Bep8X2RRjuoyuxkKsKxFuQ` (set `REDDIT_OAUTH_CLIENT_ID`). Accepted trade-off: this is outside the token's intended use (plausible ToS risk) and Reddit can rotate the client id — hence it's config-overridable.
- **Prisma pinned to 6.x.** npm's latest is Prisma 7, which drops `url` from the datasource block and requires a `prisma.config.ts` + driver adapter passed to `PrismaClient`. For a scaffold that's unnecessary ceremony, so we pinned `prisma`/`@prisma/client` to `^6` (6.19.3) — classic `url = env("DATABASE_URL")` schema and adapter-free `PrismaService`. Revisit if a v7 feature is needed.
- **Angular Material 20 theme configured by hand.** `ng add @angular/material` installed the packages but its theming schematic errored (`azure/blue` "primary" bug) before writing config, and it did not add `@angular/animations`. We wired the theme in `styles.scss` (`mat.theme(...)`), animations via `provideAnimationsAsync()`, and installed `@angular/animations` manually.
- **Frontend versions:** Angular 20 / Material 20 (newer than the "18+" floor in CLAUDE.md) — that's what the CLI installed; standalone-components + signals patterns apply.
- **Auth = access + refresh tokens, admin-managed accounts.** No public registration: the seed script creates the first admin; admins create shillers via `POST /api/users` (role forced to `shiller` server-side). Access token 15m, refresh 7d. Refresh tokens are rotated on use and stored only as SHA-256 hashes in `RefreshToken`, so the DB can revoke/expire them without ever holding the raw token. Logout revokes only the presented (current-session) token. `bcrypt` at 12 rounds via `common/crypto/password.util.ts`.
- **Frontend session in `localStorage`.** Tokens + user cached under `dash.*` keys; the interceptor refreshes once on a 401 and force-logs-out to `/login` if that fails. Chosen for simplicity over httpOnly cookies for this internal tool.
- **DB = Supabase Postgres via the session pooler (port 5432).** That URL works for `prisma migrate`. Migrations applied directly from the dev machine.
- **UI follows the "Signal." prototype, not Material chrome.** `Dashboard_UI_Prototype.html` is the visual spec. The app shell is hand-built (custom dark sidebar/topbar), and shared visual primitives (cards, tables, pills) are global classes in `styles.scss`; Angular Material is kept only where it earns its keep (dialogs, form fields, menus, snackbars, the date-range picker). Charts are **inline SVG** — no ng2-charts/Chart.js dependency (it was listed in status but never actually installed).
- **Role → view mapping.** Dash has only `admin` + `shiller`. `admin` gets the prototype's manager/marketing overview (all accounts across shillers + Team & Access); `shiller` gets the employee overview (own accounts only). Enforced server-side in `getDashboard` (scope by role) and by `roleGuard('admin')` on `/team`.
- **One dashboard aggregation endpoint.** `GET /api/reddit/dashboard` computes per-account weekly comment counts + karma server-side (through the shared 60/min throttle) so the browser never fans out N Reddit calls. Cost is ~2 Reddit calls/account, run sequentially; caching is deferred until account volume warrants it.
- **Weeks are Monday-based.** `startOfWeek` = Monday 00:00 local; `this-week` filters Mon→now, `last-week` the prior Mon–Sun. The topbar label always shows the full Mon–Sun span (e.g. "Jul 6 – Jul 12, 2026").

## Known Issues

- **`REDDIT_REFRESH_TOKEN` must be set in `backend/.env`** (agent-write-blocked — user adds it). Extract it once from `~/.devvit/token` with:
  `node -e "const fs=require('fs');const o=JSON.parse(fs.readFileSync(process.env.USERPROFILE+'/.devvit/token'));console.log(JSON.parse(Buffer.from(o.token,'base64').toString()).refreshToken)"`
  Only `REDDIT_USER_AGENT` + `REDDIT_REFRESH_TOKEN` are required now (optionally `REDDIT_OAUTH_CLIENT_ID` if the token came from the localhost devvit login flow). `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`/`REDDIT_REDIRECT_URI` are **no longer read** and can be removed. The Reddit-API path itself is verified live; only the `.env` value + a browser click-through remain.
- **`backend/.env` and `.env.example` are agent-write-blocked.** The real `.env` was hand-created by the user with generated JWT/encryption secrets. `.env.example` still needs `JWT_ACCESS_SECRET`/`JWT_ACCESS_EXPIRES`, `JWT_REFRESH_SECRET`/`JWT_REFRESH_EXPIRES`, and `ADMIN_EMAIL`/`ADMIN_PASSWORD` added by hand if not already present.
- **Seeded admin uses a placeholder password.** `admin@dash.local` / `DashAdmin123!` — change it (re-seed after deleting the row, or add a password-change flow).
- **Frontend bundle is back under the 500 kB budget** (~398 kB initial after the redesign dropped the Material shell in favor of the custom dark shell). No budget warning currently.
- **Dashboard/History load is O(accounts) Reddit calls.** The admin dashboard makes ~2 throttled Reddit calls per account (comments + `/about`) sequentially; admin Activity History re-derives the account list from `dashboard()` and then fetches comments again (a second pass).
